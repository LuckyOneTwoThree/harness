# 会话收尾技能 (Close Chat)

> Agent 会话结束前的标准流程。确保每次会话都有记录、有状态更新、有学习捕获。

## 何时触发

- Agent 完成一个独立任务后
- Agent 准备结束会话前
- 用户说"先到这里" / "今天先这样" / "收工"

## 执行步骤

### 1. 更新 progress.md

追加本次会话记录：

```markdown
### [日期] - [会话标识]
- **目标**: [本次会话目标]
- **完成**:
  - [x] [完成项1]
  - [x] [完成项2]
- **未完成**:
  - [ ] [未完成项 + 原因]
- **发现的问题**:
  - [问题描述 + 是否已解决]
- **下一步**:
  - [建议的下一步操作]
```

### 2. 更新 features.json

将已完成的功能标记为 `done`，阻塞的标记为 `blocked`。

### 3. 提取教训

如果本次会话遇到了以下情况，记录到 `.harness/rules/` 对应文件：
- 犯了不该犯的错 → 记录到 rules 对应文件
- 发现了新的编码规范 → 补充到 rules
- 发现了安全问题 → 补充到 security.md

### 4. 运行收尾脚本

```bash
bash .harness/skills/close-chat/run.sh
```

### 5. 通知协作者（如有）

如果改动影响其他人的模块，在 progress.md 中标注并通知。
